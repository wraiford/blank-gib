Architectural Details for Frontend Innovation

  • DLT Redux: Each ibgib is like a monotonoically-increasing mini-repo. They're like a Merkle-DAG based append-only log. Each component has a backing model of one of these ibgibs. So we get a redux-like design but tied to a decentralized, DLT-based data model.
  • Agent-backed: Agents suck at complex projects - but append-only because we haven't shifted our "version control" paradigm. The front end itself is an interface to high-velocity, ratcheting version control akin to a movie being a high framerate of individual pics. Agents and humans collaborate with agents and humans in a new paradigm.
  • Command based: Currently we have a central commanding architecture that plugs into the agentic function calling. There are no `eval` calls.
  • Dynamic UX: We have started down the road of an agent-backed, canvas-based, dynamic rendering engine. The shell itself is HTML. Agents+Humans generate artifacts that evolve in the unique ibgib version control and propagate throughout the distributed data space.
